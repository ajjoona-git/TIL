# AI 서비스 배포: 인프라 구성과 모니터링

## API 프레임워크
### Fast API

- python 기반의 웹 API 프레임워크
- 타입을 활용한 자동 문서 생성, 데이터 검증, 직렬화 기능 등
- 비동기 프로그래밍을 네이티브로 지원한다.

## 모니터링 시스템
### Prometheus

- 시계열 DB기반의 오픈소스 모니터링 시스템
- 주기적으로 서비스의 지표들을 수집, 저장하며 이를 조회할 수 있는 쿼리 언어 (PromQL)을 제공한다.
- Grafana와 함께 사용되어 수집된 지표들을 시각화한다.

### Grafana

- 데이터 소스로부터 지표를 가져와 대시보드 형태로 시각화하는 오픈소스
- GUI를 통해 그래프, 게이지, 히트맵, 테이블 등 다양한 형태의 시각화 패널 구성
- 이메일, Slack 등으로 이상 알림 수신
- 실시간 모니터링이 필요한 인프라, 어플리케이션 성능, 서비스 비즈니스 지표 등을 한 눈에 파악할 수 있는 대시보드를 제공한다.

## 컨테이너 기술
### Docker

- 어플리케이션과 그 실행에 필요한 모든 의존성을 하나의 가상 컨테이너로 패키징하여 어떤 환경에서든 동일하게 실행할 수 있게 해주는 컨테이너 플랫폼
- 환경 차이로 인한 문제를 해결하여 일관된 배포를 가능하게 한다.
- 컨테이너는 OS 수준에서 격리되어 실행되므로 리소스를 효율적으로 사용하면서도 어플리케이션 간의 충돌을 방지하고, Docker Hub 같은 레지스트리를 통해 이미지를 쉽게 공유하고 배포할 수 있다.
